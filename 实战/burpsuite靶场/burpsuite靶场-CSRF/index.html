7.通过 cookie 刷新绕过 SameSite Lax

  1. 在监视 Burp 中的代理历史记录的同时再次测试对自己的攻击。
  2. 出现提示时，单击页面。这会触发 OAuth 流程并向您发出一个新的会话 cookie。5 秒后，请注意已发送 CSRF 攻击并且请求POST /my-account/change-email包含您的新会话 cookie。
  3. 转到您的帐户页面并确认您的电子邮件地址已更改。
  4. 更改您的 exploit 中的电子邮件地址，使其与您自己的不匹配。
  5. 将漏洞传递给受害者以解决实验室问题。

  2. 将请求发送到 Burp Repeater。请注意，如果更改 Referer HTTP 标头中的域，请求将被拒绝。
