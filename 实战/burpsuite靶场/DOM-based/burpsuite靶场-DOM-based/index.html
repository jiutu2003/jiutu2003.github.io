6.利用 DOM 破坏来启用 XSS

打开网页，转到一个博客文章

我们发表一个评论，其中评论内容可以造成XSS

发现并没有执行

我们发表第二条评论

在文章的页面导入 JavaScript 文件loadCommentsWithDomPurify.js

该defaultAvatar对象是使用这种包含逻辑OR运算符和全局变量的危险模式实现的。这使得它容易受到DOM 破坏的攻击。

可以使用锚标记破坏此对象。

创建两个具有相同 id 的锚点会导致它们被分组到一个 DOM 集合中。第一个锚点中的属性name= "avatar"，它将使用href的内容破坏属性avatar。

该站点使用 DOMPurify 过滤器来尝试减少基于 DOM 的漏洞。但是，DOMPurify 允许使用cid:不对双引号进行 URL 编码的协议。这意味着可以使用编码的双引号。

上述注入将导致 defaultAvatar 变量在下次加载页面时被分配破坏属性 {avatar: ‘cid:”onerror=alert(1)//’}

当发布第二个帖子时，浏览器会使用新破坏的全局变量，该变量会在事件处理程序中走私有效负载onerror并触发alert().
