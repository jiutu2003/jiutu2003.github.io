凯撒密码

凯撒密码（caesar cipher），或称凯撒加密、凯撒变换、变换加密，是一种最简单且最广为人知的加密技术。凯撒密码是一种替换加密技术，明文)中的所有字母都在字母表上向后（或向前）按照一个固定数目进行偏移后被替换成密文。例如，当偏移量是3的时候，所有的字母A将被替换成D，B变成E，以此类推，当移动到最后一位时，会从头开始循环。

str_rot13 () 函数逆向解密

str_rot13()函数是一个字符串函数，用于使用ROT13编码技术对字符串进行编码(或解码)。ROT13编码技术仅用于编码字母，不编码数字和其他特殊字符。 在ROT13编码技术中，每个字母移位13在字母表中都处于位置。它接受字符串并通过ROT13编码技术返回编码的字符串。

CBC字节翻转攻击

CBC加密模式

  • Plaintext:明文
  • IV:初始化向量
  • Ciphertext:加密后的数据
  • Key:密钥

解密过程转化成文字来叙述：

先将Plaintext分组（常见以16字节为一组）
生成初始化向量IV和密钥Key（IV不需要保密，但一定要不可预测）
将IV和明文1组异或
将3中所得结果用Key进行加密，得到密文1组
将密文1组当作新IV与明文2组进行异或处理
将5中的结果用Key进行加密，得到密文2组
重复上述步骤，最后将IV和所有的密文分组拼接起来，得到最终密文

CBC解密模式

解密过程转化成文字来叙述：

  1. 从密文中提取出IV,并将密文分组
  2. 用Key将密文1组解密
  3. 2中解密的结果与IV异或，获得明文1组
  4. 密文1组成为新一轮的IV
  5. 重复上述步骤，将所有明文分组拼接后，得到最终明文

CBC字节反转攻击原理

根据解密过程，我们可以得出，当我们修改密文1的第N个字节时，后面的解密过程第N个字节进行异或运算时，得到的明文的第N个字节会受到影响

XOR即异或运算，异或运算中，相同为0，不同为1。异或运算有个特性，是可以自定义异或结果的。

假设 A ^ B = C，则

B = A ^ C，

当人为修改 A = A ^ C时，

A ^ B = A ^ C ^ B = B ^ B = 0 = C；

当人为修改 A = A ^ C ^ x (x为任意数值)时，

A ^ B = A ^ C ^ x ^ B = B ^ B ^ x = x = C

如：

密文1[N] = A，密文2[N] = B，明文[N] = C

即 A ^ B = C，那么 B = A ^ C，

当人为修改 A = A ^ C 时，

A ^ B = A ^ C ^ B = B ^ B = 0 = C，那么明文[N] = 0，

当人为修改 A = A ^ C ^ 1 时，

A ^ B = A ^ C ^ 1 ^ B = B ^ B ^ 1 = 1 = C，那么明文[N] = 1，

这样就可以控制明文第N个字节了
